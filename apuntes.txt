PETICIONES GET

GET     -> solicita una representación de un recurso específico. Las peticiones que usan el método GET sólo deben recuperar datos.
POST    -> se utiliza para enviar una entidad a un recurso en específico, causando a menudo un cambio en el estado o efectos secundarios en el servidor.
PUT     -> reemplaza todas las representaciones actuales del recurso de destino con la carga útil de la petición.
DELETE  -> borra un recurso en específico.
PATCH   -> es utilizado para aplicar modificaciones parciales a un recurso.

EXPRESS BASADO EN CLASES / FACTORY METHOD

- Para crear el server se crea una carpeta llamada 'models' donde estaran todas las clases en este caso 'server.js'

- Las propiedades van en el constructor

- Las rutas se haran con el metodo routes. 

- Para devolver un status se debe colocar res.status(codigo_http)

- Dependiendo del tipo de peticion es que llamara a los diferentes endpoints

- CORS middleware que proteje el servidor y que permite que solo ciertas paginas accedan a este

- SEPARAR RUTAS DE LA CLASE SERVIDOR -> dentro de la carpeta creada routes se pondran los controladores de los endpoints

- Se colocan las diferentes rutas de la app como propiedad de la clase server

- Router solamente debe tener la direccion y proteccion de la ruta, el controlador deberia estar en un archivo independiente

- Se debe de configurar un middleware para avisar que la informacion que viene al backend se recibira en formato JSON
express.json() -> parsea las peticiones entrantes en el body
intercepta la solicitud antes de que llegue a los controladores de ruta y analiza los datos JSON en un objeto JavaScript

PARAMETROS DE SEGMENTO 

- Son valores que forman parte de la URL de una solicitud HTTP y que se utilizan para transmitir información específica al
servidor. Estos parámetros son segmentos de la URL que se encuentran después de la ruta base y se utilizan para identificar
recursos o realizar operaciones específicas en el servidor.

- Se colocan en el archivo de las rutas -> router.put('/:id', usersPut); -> express lo parsea automaticamente en una propiedad del objeto req.params

- Para recuperar los parametros de la query se tiene que hacer con req.query (!!!!)
